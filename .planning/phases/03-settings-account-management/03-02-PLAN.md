Create four dialog components in `frontend/components/settings/`. Each follows the AddSubDialog pattern: `"use client"`, react-hook-form + zodResolver, useState for `submitting`, form.reset() on dialog close, toast feedback via sonner.

**All dialogs share this structure:**
- Props: `{ open: boolean; onOpenChange: (open: boolean) => void; ... }`
- `const form = useForm<FormValues>({ resolver: zodResolver(schema), defaultValues: {...} })`
- `const [submitting, setSubmitting] = useState(false)`
- `onOpenChange` handler: when `open` becomes false, call `form.reset()`
- Submit handler: `setSubmitting(true)`, try/catch API call, `toast.success(...)` or `toast.error(...)`, `setSubmitting(false)`, close dialog on success
- Footer: dismiss button (ghost variant) + submit button (primary variant), submit disabled when `submitting`
- Loader icon from lucide-react shown in submit button during submission



**EditNameDialog.tsx** (per D-04, D-05):

Props: `{ open, onOpenChange, currentName: string, onSuccess: () => void }`

Zod schema:
```typescript
const schema = z.object({
	name: z.string().min(1, "Name is required").max(100)
})
```

- Default value: `{ name: currentName }` — use `useEffect` to reset form when `currentName` changes or dialog opens: `form.reset({ name: currentName })` when `open` becomes true
- Dialog title: `Update Display Name`
- Input: single text input for name, `FormLabel` = "Display name"
- Submit button text: `Save Name` (while submitting: `Saving...` + Loader icon)
- Dismiss button text: `Discard Changes` (variant ghost)
- On submit: call `updateName(data.name)`, then `toast.success("Display name updated")`, call `onSuccess()`, close dialog
- On error: `toast.error("Failed to update display name. Please try again.")`



**EditEmailDialog.tsx** (per D-07, D-08, D-09):

Props: `{ open, onOpenChange, currentEmail: string, onSuccess: () => void }`

Zod schema:
```typescript
const schema = z.object({
	new_email: z.string().email("Please enter a valid email address"),
	password: z.string().min(1, "Password is required")
})
```

- Default values: `{ new_email: "", password: "" }`
- Dialog title: `Update Email Address`
- Two inputs: "New email address" and "Current password" (type="password")
- Submit button text: `Update Email` (while submitting: `Updating...` + Loader)
- Dismiss button text: `Discard Changes` (variant ghost)
- On submit: call `updateEmail(data.new_email, data.password)`
- On success: `toast.success("Email updated")`, call `onSuccess()`, close dialog
- On error: Parse error message from the thrown Error:
  - If message contains "Incorrect password" -> set form error on password field: `form.setError("password", { message: "Incorrect password" })`
  - If message contains "already exists" -> set form error on new_email field: `form.setError("new_email", { message: "Email already in use" })`
  - Otherwise: `toast.error("Failed to update email. Please try again.")`



**ChangePasswordDialog.tsx** (per D-10, D-11):

Props: `{ open, onOpenChange }`

Zod schema:
```typescript
const schema = z.object({
	current_password: z.string().min(1, "Current password is required"),
	new_password: z.string().min(8, "Password must be at least 8 characters"),
	confirm_password: z.string().min(1, "Please confirm your password")
}).refine(data => data.new_password === data.confirm_password, {
	message: "Passwords do not match",
	path: ["confirm_password"]
})
```

- Default values: `{ current_password: "", new_password: "", confirm_password: "" }`
- Dialog title: `Change Password`
- Three inputs: "Current password", "New password", "Confirm new password" (all type="password")
- Submit button text: `Change Password` (while submitting: `Changing...` + Loader)
- Dismiss button text: `Discard Changes` (variant ghost)
- On submit: call `changePassword(data.current_password, data.new_password)`
- On success: `toast.success("Password changed")`, close dialog. No logout (per D-11).
- On error: If message contains "incorrect" (case-insensitive) -> `form.setError("current_password", { message: "Incorrect password" })`. Otherwise: `toast.error("Failed to change password. Please try again.")`



**DeleteAccountDialog.tsx** (per D-12, D-13, D-14):

Props: `{ open, onOpenChange }`

This dialog uses a slightly different pattern — the confirm button is disabled until input === "DELETE".

Zod schema (optional — can also use plain controlled state):
```typescript
const schema = z.object({
	confirmation: z.literal("DELETE", {
		errorMap: () => ({ message: "Type DELETE to confirm" })
	})
})
```

OR simpler approach with `useState`:
```typescript
const [confirmation, setConfirmation] = useState("")
const canConfirm = confirmation === "DELETE"
```

Use whichever approach is cleaner. The simpler useState approach avoids zod complexity for a single literal check.

- Dialog title: `Delete Account`
- Dialog body text: `This action is permanent and cannot be undone. All your subscriptions, categories, and account data will be permanently deleted.`
- Input: text input with `aria-label="Type DELETE to confirm"` and placeholder `Type DELETE to confirm`
- Submit button text: `Delete My Account` (while submitting: `Deleting...` + Loader). Use `variant="destructive"`. Disabled when confirmation !== "DELETE" or submitting.
- Dismiss button text: `Keep My Account` (variant ghost)
- On submit: call `deleteAccount()`, then call `logout()`, then `toast.success("Account deleted")`, then redirect: `router.push("/sign-in")` using `useRouter` from `next/navigation`
- On error: `toast.error("Failed to delete account. Please try again.")`
- On dialog close: reset confirmation to empty string

Import `useRouter` from `next/navigation` for the redirect.
