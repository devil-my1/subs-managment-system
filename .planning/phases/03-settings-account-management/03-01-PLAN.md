- Error handling: `try/except HTTPException: raise / except Exception as e:` pattern — log with `auth_logger.exception("name_update_error", ...)`, raise 500 with "Could not update name. Please try again later."

**PATCH /auth/update-email endpoint** (add after update-name):
- Decorator: `@router.patch("/update-email", response_model=Message)`
- Parameters: `payload: UpdateEmail, user: User = Depends(get_current_user), db: AsyncSession = Depends(get_db)`
- Body:
  1. Fetch fresh user from DB (CRITICAL: cache does not store password_hash per SEC-03, so must query DB directly)
  2. Verify password: `if not verify_password(payload.password, db_user.password_hash): raise HTTPException(status_code=401, detail="Incorrect password")`
  3. Check email uniqueness: `existing = await db.execute(select(User).where(User.email == payload.new_email))` — if found, raise 409 "An account with this email already exists"
  4. Update: `db_user.email = payload.new_email`, commit, `await cache_delete_key(f"user:{db_user.id}")`
  5. Log: `auth_logger.info("email_updated", extra={"user_id": str(db_user.id), "new_email": payload.new_email})`
  6. Return `{"detail": "Email updated"}`
- Error handling: Same try/except pattern, log "email_update_error", raise 500 "Could not update email. Please try again later."

async def test_update_name(client):

    r = await client.post("/auth/register", json={"email": email, "password": password, "name": name})
