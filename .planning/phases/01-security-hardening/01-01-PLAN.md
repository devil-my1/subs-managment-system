  <behavior>

    - verify_password("testpassword", hash_password("testpassword")) returns True

  <action>
Completely rewrite backend/src/core/security.py. The new file must:

1. Remove ALL imports of passlib and jose. Import jwt (PyJWT) and bcrypt directly.

2. hash_password(password: str) -> str:
   - Calls bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt(rounds=12))
   - The result is bytes — decode to str before returning: .decode("utf-8")
   - Return type is str (same as before — DB column is VARCHAR)

3. verify_password(password: str, password_hash: str) -> bool:
   - Calls bcrypt.checkpw(password.encode("utf-8"), password_hash.encode("utf-8"))
   - Both arguments must be bytes — encode both from str
   - Returns bool (same as before)
   - This is compatible with existing hashes in the DB: passlib used bcrypt $2b$ prefix, direct bcrypt reads the same prefix.

4. create_access_token(subject: str) -> str:
   - Builds payload: {"sub": subject, "iat": int(now.timestamp()), "exp": int(exp.timestamp())}
   - Calls jwt.encode(payload, settings.JWT_SECRET, algorithm=settings.JWT_ALG)
   - PyJWT 2.x returns str for HS256 — same as python-jose

5. decode_token(token: str) -> dict:
   - Calls jwt.decode(token, settings.JWT_SECRET, algorithms=[settings.JWT_ALG])
   - Catches jwt.exceptions.InvalidTokenError (base class covers expired, decode errors, etc.)
   - Re-raises as ValueError("Invalid token") — same contract as before so deps.py is unaffected

Imports at top of file:
  from datetime import datetime, timedelta, timezone
  import bcrypt
  import jwt
  from jwt.exceptions import InvalidTokenError
  from src.core.config import settings

The public API (hash_password, verify_password, create_access_token, decode_token) must remain identical in name and signature — no other files need changes for this plan.
